Getting Started with Claude Code

Claude Code is Anthropic's agentic coding tool. It reads your codebase, edits files, runs commands, and integrates with your terminal, IDE, browser, and apps.

Overview

What Claude Code is and why we use it

Claude Code is an agentic coding tool that reads your codebase, edits files, and runs commands. It works in your terminal, IDE, browser, and as a desktop app. Unlike simple code completion, Claude Code operates in an agentic loop ‚Äî it gathers context, plans, acts, and verifies its own work.

How Claude Code Works

The agentic loop and available tools

Claude Code operates through an intelligent three-phase agentic loop that adapts to your task. Unlike traditional code assistants that generate one-shot responses, Claude gathers context, acts, and verifies its own work ‚Äî iterating until the task is complete.

üîç

1. Gather Context

Read/search files, analyze errors, review git state, load CLAUDE.md instructions

‚ö°

2. Take Action

Edit files, run commands, execute git operations, invoke external tools via MCP

‚úÖ

3. Verify Results

Run tests, analyze output, course-correct, iterate until the task is complete

Available Tools

Context Management

Claude has a context window ‚Äî a limit on how much information it can hold at once. As conversations grow long:

  • Older tool outputs are automatically compressed first
  • Conversation history gets summarized if needed
  • CLAUDE.md instructions and important context are preserved
  • Use /compact to manually compress the conversation
  • Use /clear to reset and start fresh between unrelated tasks

Permission System

Claude asks for your approval before performing potentially impactful actions. You can configure the level of autonomy:

Common Workflows

How to use Claude Code for everyday tasks

Explore ‚Üí Plan ‚Üí Code ‚Üí Commit

The officially recommended workflow for non-trivial tasks:

Explore (Plan Mode)

Start with Shift+Tab to enter Plan Mode. Ask Claude to read and understand the relevant code before making changes.

prompt
> read src/auth/ and explain how session handling works

Plan

Ask for a detailed implementation plan. Review it, adjust if needed.

prompt
> create a plan to add OAuth2 support using the existing auth patterns

Implement

Switch back to Normal Mode (Shift+Tab) and let Claude execute the plan.

prompt
> implement the plan. write tests and run them after each change

Commit

prompt
> commit the changes with a descriptive message and create a PR

Understand a Codebase

prompts
> give me an overview of this codebase
> explain the main architecture patterns
> what are the key data models?
> how does authentication work?
> trace the flow from a user clicking "Buy" to the order being saved

Fix Bugs

prompts
> the login form throws a TypeError when email is empty - find and fix it
> users report a blank screen after failed login. check token refresh in src/auth/
> fix the race condition in the checkout flow where inventory isn't checked before payment

Write and Fix Tests

prompts
> write comprehensive tests for the CartService class, aim for >90% coverage
> find untested functions in the notification module and add edge case tests
> run the test suite and fix all failures

Refactor Code

prompts
> refactor fetchUserData to use async/await instead of .then() chains
> migrate our Express routes from JavaScript to TypeScript with proper type definitions
> find deprecated API usage across the project and suggest modern alternatives

Git & PRs

prompts
> commit my changes with a descriptive message
> create a new branch called feature/oauth2
> create a PR with a summary of what changed and why
> help me resolve the merge conflicts in src/api/users.ts

Pipe & Script (Headless)

bash

Context Management

Your most important resource

The context window fills fast and performance degrades when it's overloaded. Keep it clean:

  • Use /clear between unrelated tasks
  • Use /compact to compress long conversations
  • Run /clear after 2+ failed correction attempts ‚Äî start fresh with a better prompt
  • Use subagents for exploratory work (they get separate context)
  • Put persistent rules in CLAUDE.md, not in conversation
  • Track context usage with /context

Always Use Version Control

bash
# Create a branch before big changes
git checkout -b claude/add-auth-module
claude "implement JWT authentication middleware"
# Review changes, then merge when satisfied

Review Everything

Claude is powerful but not perfect. Always check for:

  • Logic errors or incorrect assumptions
  • Security vulnerabilities (SQL injection, XSS, hardcoded secrets)
  • Performance issues (N+1 queries, unnecessary loops)
  • Adherence to your team's conventions

Pitfalls to Avoid

PitfallFix
Kitchen-sink sessions (mixing unrelated tasks)/clear between different tasks
Correcting Claude repeatedly (context gets muddled)/clear and rewrite a better initial prompt
Over-specified CLAUDE.mdKeep it short ‚Äî less is more. Prune ruthlessly
Trust-then-verify gapAlways include tests or screenshots as verification
Infinite explorationScope tasks narrowly or use subagents
Security: Be cautious about prompt injection. Don't paste untrusted content directly into Claude without reviewing it. Claude has built-in protections, but awareness is key.

CLAUDE.md

Project memory and team standards

CLAUDE.md is a markdown file that Claude reads at the start of every session. It's your way to give Claude persistent context about your project, coding standards, and preferences. Run /init to create one.

Where to Put It

LocationScopeGit
~/.claude/CLAUDE.mdGlobal ‚Äî all projects, all sessionsNot in git
./CLAUDE.mdProject ‚Äî shared with teamCheck into git
./CLAUDE.local.mdPersonal project prefsAdd to .gitignore
./src/CLAUDE.mdSubdirectory ‚Äî auto-loaded for that folderCheck into git

What to Include

markdown
# CLAUDE.md

## Tech Stack
- TypeScript strict mode, React 18, Next.js 14
- Tailwind CSS for styling
- Vitest for testing (not Jest)
- pnpm (not npm or yarn)

## Code Style
- ES modules (import/export), not CommonJS
- Functional components with hooks
- Destructured imports: `import { foo } from 'bar'`
- Keep components under 200 lines

## Commands
- Build: `pnpm build`
- Test: `pnpm test`
- Lint: `pnpm lint`
- Type check: `pnpm typecheck`

## Architecture
- API routes in src/api/
- Components in src/components/
- Business logic in src/services/
- All DB access through Supabase Client (@supabase/supabase-js)

## Conventions
- Branch naming: feature/*, bugfix/*, hotfix/*
- Commit messages: conventional commits (feat:, fix:, refactor:)
- PR descriptions must include "What" and "Why" sections

What NOT to Include

  • Standard language conventions (Claude already knows them)
  • Full API documentation (link to it instead)
  • Things that change frequently (version numbers, etc.)
  • Self-evident practices ("write clean code")
Keep your CLAUDE.md concise. Long files waste context tokens every session. Aim for under 100 lines.

Hooks

Automate actions before or after Claude acts

Hooks run deterministic commands at specific points in Claude's workflow ‚Äî like linting after edits or blocking changes to certain directories.

Configuring Hooks

Run /hooks in a session, or configure in .claude/settings.json:

json
{
  "hooks": {
    "afterEdit": [
      { "command": "eslint --fix $FILE", "description": "Auto-fix lint issues" }
    ],
    "beforeCommit": [
      { "command": "pnpm typecheck", "description": "Type-check before committing" }
    ]
  }
}

Available Hook Points

  • afterEdit ‚Äî runs after any file is modified
  • beforeCommit ‚Äî runs before git commits
  • afterCommit ‚Äî runs after git commits
  • onSessionStart ‚Äî runs when a session begins

Subagents

Specialized AI assistants within a session

Subagents are specialized assistants that run in their own context window with specific tools and permissions. They help you:

  • Preserve context ‚Äî keep exploration out of your main conversation
  • Enforce constraints ‚Äî limit which tools a subagent can use
  • Reuse configs ‚Äî same subagent across projects
  • Specialize ‚Äî dedicated agents for testing, docs, security review
  • Control costs ‚Äî route tasks to faster, cheaper models like Haiku

Creating a Subagent

.claude/agents/security-reviewer.md
---
name: security-reviewer
description: Review code for security vulnerabilities
model: sonnet
tools:
  - Read
  - Grep
  - Glob
---

You are a security-focused code reviewer. Analyze code for:
- SQL injection, XSS, CSRF vulnerabilities
- Authentication and authorization issues
- Hardcoded secrets or credentials
- Insecure dependencies

Use /agents to manage subagents. Claude automatically delegates when a task matches a subagent's description.

Agent Teams

Multiple agents working in parallel

While subagents work within a single session, agent teams coordinate across separate sessions. A lead agent distributes work and merges results.

  • Parallel processing ‚Äî multiple agents work on different parts simultaneously
  • Separate git worktrees ‚Äî each agent works in isolation
  • Lead agent coordinates ‚Äî distributes tasks, reviews output, merges
  • Useful for large refactors, multi-service changes, comprehensive testing
Agent teams are best for tasks that are naturally parallelizable. For sequential tasks, use regular sessions or subagents instead.

Permissions

Control what Claude can and can't do

Claude asks permission before running commands or editing files. You can pre-approve safe commands:

.claude/settings.json
{
  "permissions": {
    "allow": [
      "Bash(npm test)",
      "Bash(npm run lint)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(pnpm *)",
      "Read",
      "Glob",
      "Grep"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)"
    ]
  }
}

Permission Modes

Toggle with Shift+Tab or --permission-mode flag:

  • Normal: Ask for each edit and command
  • Auto-Accept Edits: Edit files freely, ask for commands
  • Plan Mode: Read-only. Creates a plan for approval before any changes
--dangerously-skip-permissions bypasses all permission prompts. Only use this in sandboxed CI environments, never locally.

Security

Data privacy and safety

  • Code context is sent to Anthropic's API for processing
  • Anthropic does not train on your code by default
  • Team and Enterprise plans have additional data privacy guarantees
  • Claude respects .gitignore ‚Äî ignored files are never sent
  • Permission system prevents unauthorized destructive actions
  • Sandboxing available for OS-level isolation
  • All API communication is encrypted over HTTPS
Prompt injection: Be aware that content in files could attempt to manipulate Claude. Don't blindly paste untrusted content. Claude has protections but awareness helps.

Checkpointing

Undo anything ‚Äî safely

Claude automatically creates checkpoints before making changes. You can rewind to any checkpoint to restore both code and conversation state.

How to Rewind

  • Terminal: Press Esc twice, or use /rewind
  • VS Code: Hover over a message and click the rewind icon
  • Rewind restores: code changes, conversation state, and file system
  • You can fork from a checkpoint to try a different approach
Checkpoints persist across sessions but are not a replacement for git. Always commit important work.

The 2 Specialized Agents

supabase-schema-architect

A dedicated AI agent that focuses on database design, migrations, and Row-Level Security (RLS) policies. It analyzes your requirements and generates production-ready schemas with optimal performance.

  • Designs normalized database schemas from plain-English descriptions
  • Generates RLS policies for multi-tenant, role-based, and hierarchical access
  • Creates migration files with proper foreign keys, indexes, and constraints
  • Audits existing schemas for security gaps and performance issues
  • Handles complex patterns: multi-tenant SaaS, time-based access, recursive sharing
Usage
# Claude Code automatically delegates to this agent when you ask things like:
> Design a database schema for a project management app with organizations,
  projects, tasks, and comments. Add RLS so users only see their org's data.

> Add an orders table with user_id, total_cents, and created_at fields.
  Include proper RLS policies and generate the migration file.

> Audit all my RLS policies and find any tables with missing or
  overly permissive access rules.
supabase-realtime-optimizer

Specializes in WebSocket optimization and real-time performance. It monitors connections, optimizes subscriptions, and ensures scalable real-time features.

  • Optimizes Supabase Realtime channel subscriptions for performance
  • Monitors WebSocket connection health and diagnostics
  • Designs efficient Presence, Broadcast, and Postgres Changes patterns
  • Identifies and fixes subscription leaks and connection bottlenecks
  • Scales real-time features for production workloads
Usage
# Claude Code delegates to this agent for real-time tasks:
> Optimize my Realtime subscriptions ‚Äî I have 12 channels and
  the WebSocket keeps disconnecting under load.

> Build a real-time notification system using Postgres Changes
  that scales to 10k concurrent users.

  with minimal bandwidth usage.

MCP Server Configuration

The MCP server provides direct Supabase API integration through the Model Context Protocol. After installation, your .mcp.json is configured automatically.

Supabase provides an official Remote MCP Server ‚Äî the simplest way to connect. No API keys or tokens needed; authenticate via browser.

.mcp.json ‚Äî Remote MCP
{
  "mcpServers": {
    "supabase": {
      "url": "https://mcp.supabase.com/mcp"
    }
  }
}

Then authenticate in Claude Code:

Terminal
# Add via CLI
claude mcp add --transport http supabase https://mcp.supabase.com/mcp

# Authenticate (opens browser login)
# Type /mcp in Claude Code and follow the login flow

For project-scoped or read-only configurations:

.mcp.json ‚Äî Project-scoped & Read-only
{
  "mcpServers": {
    "supabase": {
      "url": "https://mcp.supabase.com/mcp?project_ref=YOUR_PROJECT_REF&read_only=true"
    }
  }
}

URL parameters for fine-grained control:

ParameterDescription
read_only=trueExecutes queries as a read-only Postgres user
project_ref=<id>Restricts access to a specific Supabase project
features=<groups>Enables only specified tool groups (e.g., database,docs)

Adding Supabase Skills to Claude Code

Supabase has released an open-source set of Postgres Best Practices as Agent Skills ‚Äî a format originally developed by Anthropic to give AI agents domain expertise. These skills teach Claude Code (and other agents) how to write correct, production-ready Postgres code instead of relying on training data alone.

Based on the official Supabase blog post: Introducing: Postgres Best Practices for AI Agents.

Why Agent Skills Matter

AI agents are good at writing code but struggle with systems they don't fully understand. Postgres has decades of features, edge cases, and performance characteristics. Without explicit guidance, agents might:

  • Generate a query that works but creates a full table scan
  • Suggest an index that slows down writes instead of speeding up reads
  • Miss Row Level Security entirely, leaving data exposed
  • Create migrations that lock tables in production
  • Exhaust connection pools from poorly managed clients

Agent Skills solve this by providing explicit, authoritative rules the agent reads on demand ‚Äî like documentation written for machines.

What's in the Skills Pack

The repository contains 30+ rules across 8 categories, prioritized by impact:

CategoryPriorityWhat It Covers
Query PerformanceCriticalWriting efficient queries, avoiding full table scans
Connection ManagementCriticalConnection pooling, client lifecycle, resource limits
Security & RLSCriticalRow Level Security policies and access control patterns
Schema DesignHighTable structure, data types, normalization decisions
Concurrency & LockingMedium-HighTransaction isolation, deadlock prevention, lock management
Data Access PatternsMediumPagination, bulk operations, query design
Monitoring & DiagnosticsLow-MediumQuery analysis, performance tracking, debugging
Advanced FeaturesLowCTEs, window functions, extensions

Each rule follows a consistent format with a clear title, priority level, explanation of why it matters, and correct vs. incorrect code examples.

Example Rule: Enable RLS for Multi-Tenant Data

Relying only on application-level filtering ‚Äî a bug or bypass exposes all data:

SQL ‚Äî Application-level filtering only
-- Relying only on application to filter
SELECT * FROM orders
WHERE user_id = $current_user_id;

-- Bug or bypass means all data is exposed!
-- Returns ALL orders:
SELECT * FROM orders;

Database-enforced RLS ensures security at the Postgres level:

SQL ‚Äî Database-enforced RLS
-- Enable RLS on the table
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;

-- Create policy for users to see only their orders
CREATE POLICY orders_user_policy ON orders
  FOR ALL
  USING (
    user_id = current_setting('app.current_user_id')::bigint
  );

-- Force RLS even for table owners
ALTER TABLE orders FORCE ROW LEVEL SECURITY;

-- For the authenticated role (Supabase specific):
CREATE POLICY orders_user_policy ON orders
  FOR ALL
  TO authenticated
  USING (user_id = auth.uid());

FAQ

Frequently asked questions

What subscription do I need?
Claude Code requires a Pro ($20/mo), Max ($100 or $200/mo), Team, or Enterprise plan. Alternatively, you can use an Anthropic Console account with API credits. The free tier does not include Claude Code.
What if I hit usage limits?
Pro has a rolling usage window ‚Äî wait and it resets. Max plans have much higher limits. Reduce consumption by using Sonnet (default) for routine tasks and reserving Opus for complex reasoning. Use /compact to save context tokens.
How do I integrate Claude Code with our repo?
Just cd into your project and run claude. It reads your codebase, respects .gitignore, and works within your existing git workflow. Add a CLAUDE.md for team standards. For CI, use the claude-code-action GitHub Action.
How is this different from GitHub Copilot or ChatGPT?
Claude Code is agentic ‚Äî it doesn't just suggest code, it reads your entire codebase, edits files, runs terminal commands, manages git, and iterates on failures. It operates in a loop (gather context ‚Üí act ‚Üí verify) rather than generating one-shot completions. It works in your actual project context, not a sandboxed snippet environment.
Is my code sent to Anthropic?
Yes, code context is sent to Anthropic's API for processing. Anthropic does not train on your data by default. Team and Enterprise plans have additional data privacy guarantees. Claude respects .gitignore and never sends ignored files.
Can Claude run destructive commands?
Claude has a permission system. It asks approval before running commands that modify your system. You can configure permission levels ‚Äî from asking every time to auto-approving safe operations. It won't force-push, delete branches, or run rm -rf without explicit approval.
What languages and frameworks does it support?
Any language and framework. Especially strong with JavaScript/TypeScript, Python, Rust, Go, Java, C/C++, Ruby, PHP. Understands React, Next.js, Django, FastAPI, Express, Rails, and more. Also works with infrastructure code (Terraform, Docker, Kubernetes).
How do I resume a previous session?
Run claude -c to continue the most recent session, or claude -r for an interactive picker. Name sessions with /rename my-task for easy resumption later: claude -r my-task.
Can I use Claude Code in a monorepo?
Yes. Claude handles monorepos well ‚Äî it navigates packages, understands workspace configs (pnpm workspaces, Turborepo, Nx, Lerna), and works across multiple packages. Use --add-dir to include additional directories.
What's the difference between subagents and agent teams?
Subagents work within a single session ‚Äî they're like specialized helpers that get their own context window. Agent teams coordinate across separate sessions with separate git worktrees, useful for large parallelizable tasks. Use subagents for most cases; agent teams for big multi-service changes.
